  SECTION 1 — SYSTEM ARCHITECTURE OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CAIE Scholar is a two-tier web application:

  • Frontend  — React 18 + Vite + shadcn/ui (served by Nginx in production)
  • Backend   — FastAPI + Gunicorn (Python 3.12, async I/O via httpx)

They communicate over HTTP/REST. The backend URL is injected at build time via
the VITE_API_URL environment variable.

Stack summary:
  Frontend:   React, Vite, Tailwind CSS, shadcn/ui, Framer Motion, TanStack Query
  Backend:    FastAPI, Gunicorn, Uvicorn workers, httpx, BeautifulSoup4, pypdf
  Deployment: Docker Compose (local), Render.com (cloud), Nginx (static serving)
  AI:         Claude claude-sonnet-4-6 API (AI notes generation, client-side call)
  Sources:    pastpapers.papacambridge.com, www.gceguide.com, local disk store

Notes:
  • Source priority: Local disk → PapaCambridge → GCE Guide
  • PapaCambridge uses direct URL construction (fast, no HTML parsing needed)
  • GCE Guide uses HTML parsing as fallback when PapaCambridge has no result
  • All PDF responses are streamed directly to the browser as application/pdf
  • Search results are cached for 1 hour (memory L1 + disk L2)
  • Downloaded PDFs are cached indefinitely in data/papers/ or data/markschemes/

  DELETE  /api/downloads/delete/{id}        Remove a saved paper
  GET     /api/downloads/stats              Storage summary

  Paper ID format:  {code}_{season}_{year}_{paper}_{type}
  Example:          0580_m_j_2023_41_qp
